Promises & Async/Await

Promises handle operations that take an unknown amount of time (like fetching data from a server). They promise to return a result (success) or an error (failure) sometime in the future.

Real-world Analogy

Ordering food at a restaurant is a Promise. You place an order and get a buzzer (the Promise). The buzzer is pending. You can do other things while waiting. Eventually, it buzzes successfully (Resolved - get food) or buzzes angrily (Rejected - out of stock).

Syntax Overview

Using async/await makes asynchronous code look synchronous:

async function fetchData() {
  try {
    const res = await fetch('/api');
    const data = await res.json();
    console.log(data);
  } catch (err) {
    console.log(err);
  }
}
