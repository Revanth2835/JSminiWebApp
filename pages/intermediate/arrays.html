Arrays & Methods (API Reference)

Arrays are list-like objects whose prototype has methods to perform traversal and mutation operations. Understanding Array methods is a critical requirement for intermediate JavaScript.

Note: Some array methods mutate the original array (like push or sort), while others return a new array entirely (like map or filter).

⚠️ Mutator Methods (Changes Original)

Array.prototype.push() / pop()

Mutates original Returns: New Length / Popped Item

push() adds one or more elements to the end of an array, returning the new length.
pop() removes the last element from an array and returns that element.

Array.prototype.unshift() / shift()

Mutates original Returns: New Length / Shifted Item

unshift() adds one or more elements to the beginning of an array.
shift() removes the first element from an array.

Array.prototype.splice()

Mutates original Returns: Array of removed items

Changes the contents of an array by removing or replacing existing elements and/or adding new elements in place.

Syntax

  • arr.splice(start, deleteCount, item1, item2...)

Array.prototype.sort() / reverse()

Mutates original

sort() sorts the elements of an array in place and returns the sorted array. The default sort order is alphabetical (converting elements to strings). To sort numbers, provide a compare function.
reverse() simply reverses the array in place.

🔍 Accessor Methods (Does NOT mutate)

Array.prototype.slice()

Returns: New Array

Returns a shallow copy of a portion of an array into a new array selected from start to end (end not included). The original array will not be modified.

Array.prototype.concat()

Returns: New Array

Used to merge two or more arrays. This method does not change the existing arrays, but instead returns a new array.

Array.prototype.join()

Returns: String

Creates and returns a new string by concatenating all of the elements in an array, separated by commas or a specified separator string.

Array.prototype.indexOf() / includes()

Returns: Integer / Boolean

indexOf() returns the first index at which a given element can be found, or -1 if not present.
includes() determines whether an array includes a certain value among its entries, returning true or false.

🔄 Iterator Methods (Callbacks)

Array.prototype.forEach()

Returns: undefined

Executes a provided function once for each array element. Great for running side-effects (like logging or modifying external variables).

Array.prototype.map()

Returns: New Array

Creates a new array populated with the results of calling a provided function on every element in the calling array. Indispensable for transforming data securely.

Array.prototype.filter()

Returns: New Array

Creates a shallow copy of a portion of a given array, filtered down to just the elements from the given array that pass the test implemented by the provided function.

Array.prototype.reduce()

Returns: Single Value

Executes a user-supplied "reducer" callback function on each element of the array, passing in the return value from the calculation on the preceding element. The final result is a single value.

Syntax

  • arr.reduce((accumulator, currentValue) => { ... }, initialValue)

Array.prototype.find() / findIndex()

Returns: Element / Integer

find() returns the first element in the provided array that satisfies the provided testing function.
findIndex() returns the index of the first element that satisfies the function.

Array.prototype.some() / every()

Returns: Boolean

some() tests whether at least one element in the array passes the provided function.
every() tests whether all elements pass the provided function.
